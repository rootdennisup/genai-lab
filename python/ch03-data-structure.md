# 数据结构

Python 提供了几种非常高效的高级数据结构，主要用于存储和组织数据。

## 1 核心内置容器

### 1.1 列表（List）
可变的有序序列，支持索引访问和切片操作。它非常灵活，可以用来实现堆栈（后进先出）或队列（先进先出）。

list 常用方法：

- ``list.append(x)``：列表末尾追加一个元素，相当于 a[len(a):] = [x]。
- ``list.insert(i, x)``：在指定位置插入一个元素。
    - i：准备插入到其前面的那个元素的索引
    - 示例 a.insert(0, x) 在列表开头插入x
- ``list.remove(x)``：删除列表中值为 x 的第一个元素。若不存在，则返回一个错误。
- ``list.pop([i])``：从列表的指定位置移除元素，并将其返回。若没有指定索引，a.pop()返回最后一个元素，且元素从列表中被移除。
- ``list.sort()``：对列表中的元素进行排序。


#### 使用 list 实现堆栈

列表提供了一些方法，如：append()、 pop()非常适合栈操作（LIFO） 

栈操作：
- **压入（Push）**: 使用 append() 将一个元素添加到栈的顶端。
- **弹出（Pop）**: 使用 pop() 移除并返回栈顶元素。
- 查看栈顶元素（Peek/Top）: 返回栈顶元素而不移除它。
- 检查是否为空（IsEmpty）: 检查栈是否为空。
- 获取栈的大小（Size）: 获取栈中元素的数量。

#### 使用 list 实现队列

在 Python 中，list 可以用作队列（queue）。
- 使用列表时，如果频繁地在列表的开头插入或删除元素，性能会受到影响，因为插入、删除操作的时间复杂度是 O(n)。
- Python 提供了 collections.deque，它是双端队列，可以在两端高效地添加和删除元素， 添加和删除操作的时间复杂度是O(1) 。

### 1.2 元组 (Tuple)
- 元组（tuple）与列表类似，区别在于元组的元素不能修改。
- 元组写在小括号 () 里，元素之间用逗号隔开。
- 通常用于存储异质的数据集或作为字典的键。


### 1.3 字典 (Dictionary)
- 一种键值对 (Key-Value) 的映射结构。它通过唯一的键来快速检索对应的值，是处理结构化数据的核心
- 一对大括号创建一个空的字典：{}。


### 1.4 集合 (Set)

一个无序且元素唯一的集合。常用于去重操作或进行交集、并集等数学运算。


## 2 特色语法与操作

特色语法与操作是让代码从“能跑”进化到“优雅”的关键。推导式和嵌套结构在处理现代 Web 框架（如 FastAPI）的数据时非常核心。

### 2.1 推导式 (Comprehensions)
- 推导式是 Python 提供的一种从一个或多个迭代器快速构建序列（如列表、字典、集合）的简洁语法。
- 它本质上是将循环（for）和条件判断（if）压缩到一行代码中的表达式。

#### 有什么好处
- **代码简洁**：减少冗余代码，原本需要 3-5 行的循环操作，现在只需一行。
- **提高可读性**：对于有经验的开发者，推导式能更清晰地表达“我想根据现有数据生成什么样的新数。
- **性能更优**：在底层，推导式的执行效率通常比手动编写 list.append() 循环略高。

#### 具体案例
```python
# 案例 1：列表推导式 - 获取 1-10 中所有偶数的平方
# 语法结构：[表达式 for 变量 in 序列 if 条件]
squares = [x**2 for x in range(1, 11) if x % 2 == 0]
print(f"偶数的平方列表: {squares}")  # 输出: [4-6]

# 案例 2：字典推导式 - 将名字列表转为 {名字: 长度} 的字典
names = ["Alice", "Bob", "Charlie"]
name_lengths = {name: len(name) for name in names}
print(f"名字长度字典: {name_lengths}") # 输出: {'Alice': 5, 'Bob': 3, 'Charlie': 7}
```


### 2.2 嵌套结构 (Nested Structures)
是将一个数据结构作为另一个数据结构的元素。例如，“列表中的列表”（二维矩阵）或“字典中的列表”（复杂的 JSON 对象）。

#### 用来干什么
- **模拟复杂业务数据**：在实际工程（如 FastAPI 应用）中，从前端接收到的数据往往是多层嵌套的 JSON。
- **精准校验**：借助 Pydantic 等库，可以对深层嵌套的每一个字段进行类型校验。
- **层级化组织**：方便表达具有归属关系的数据，如“一个用户包含多条订单，每条订单包含多个商品”。

#### 具体案例
```python
    # 嵌套结构案例：字典中嵌套列表
    user_data = {
        "username": "coder_api",
        "roles": ["admin", "developer"], # 字典的值是一个列表
        "projects": [
            {"id": 1, "name": "FastAPI 学习"}, # 列表里又嵌套了字典
            {"id": 2, "name": "Pytest 自动化"}
        ]
    }

    # 访问嵌套数据
    # 获取第一个项目的名称
    first_project_name = user_data["projects"]["name"]
    print(f"第一个项目名称: {first_project_name}")

    # 结合推导式处理嵌套结构
    # 提取所有项目的 ID 列表
    project_ids = [p["id"] for p in user_data["projects"]]
    print(f"所有项目 ID: {project_ids}")
```

## 3 工程化与数据验证
在现代 Python 工程（如使用 FastAPI 或 Pydantic）中，数据结构的使用已经演进到了类型提示 (Type Hints) 驱动的阶段：
- **数据校验**：借助 Pydantic，开发者可以定义复杂的模型，系统会自动校验输入的数据结构是否符合 list、dict 或自定义模型的要求。
- **自动序列化**：Python 的内置类型（如 str、int、float、bool、list 等）可以被自动转换为网络传输的 JSON 格式，简化了 API 的开发流程。

## 4 性能与底层支持
- 解释器优化：Python 解释器提供了丰富的标准库来操作这些数据结构。
- Rust 加速：在一些高性能库（如 Pydantic V2）中，核心验证逻辑甚至使用 Rust 编写，以极大地提升处理大型 Python 数据结构的速度。
