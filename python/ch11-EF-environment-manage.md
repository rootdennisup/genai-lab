## 1 虚拟环境
虚拟环境（Virtual Environment）是 Python 工程化中实现**确定性运行时**和**依赖管理**的基础设施。

### 1.1 核心机制
- **环境隔离** ：它是 Python 解释器的一个私有副本。通过隔离，开发者可以在不影响系统全局 Python 环境的情况下，为每个项目配置独立的依赖项。
- **依赖版本控制**：复杂工程中，不同项目可能依赖同一工具库的不同版本。虚拟环境允许在同一台机器上并存多个相互冲突的库版本，解决了版本碎片化和依赖地狱的问题。
- **标准库工具 venv**：从 Python 3.3 开始，标准库引入了 venv 模块作为创建虚拟环境的官方工具。它通过在项目目录下生成一个包含 Python 二进制文件拷贝（或符号链接）的文件夹，来构建运行时的上下文环境。
- **Shell 上下文切换（Activation）**：当用户执行“激活”脚本时，系统会修改当前终端会话的 PATH 环境变量，使得 python 和 pip 命令优先指向虚拟环境内部的路径，而非系统的全局路径。
- **无污染开发**：在构建 Web 框架（如 FastAPI）或进行自动化测试（如 pytest）时，建议在虚拟环境中进行操作，以确保开发环境的纯净和项目的可移植性。

### 1.2 基本使用
- 1.**创建项目目录**，如：python_base
- 2.**创建虚拟环境**，使用 Python 自带的 venv 模块，习惯把环境文件夹命名为 .venv
    ```bash
    python -m venv .venv
    ```
    执行完，项目目录下会多了个 .venv 目录。
- 3.**激活虚拟环境**，需要运行环境内部的激活脚本
    ```bash
    # Windows
    .venv\Scripts\activate
    ```
    成功标志：你的命令行提示符前面会出现 (.venv) 字样
- 4.**在环境内安装包**(实践)：在当前的 .venv 文件夹里安装 FastAPI，不会影响其他项目
    ```bash
    pip install "fastapi[standard]"
    ```
- 5.**退出环境** ：输入 ``deactivate`` 回到全局环境


## 2 venv 隔离机制的理解
Python 的 venv（Virtual Environment）模块是实现工程化开发中“确定性运行时”和“依赖管理”的核心基础设施。可以从如下几个维度理解：

### 2.1 物理层面：“私有副本”
venv 本质是项目目录下的一个文件夹（通常命名为 .venv），该文件夹包含了 Python 解释器的私有副本。

- **独立目录结构**：虚拟环境拥有自己独立的目录结构，包括存放脚本的 Scripts（Windows）或 bin（Linux/Unix）目录，以及存放第三方库的 Lib\site-packages 文件夹。
- **二进制文件拷贝**：在创建虚拟环境时，venv 会在环境内部生成 Python 可执行文件的拷贝或符号链接。意味着当运行虚拟环境中的 Python 时，它是一个独立的进程上下文。

### 2.2 运行时层面：PATH 环境变量拦截
虚拟环境的“激活”（Activation）是实现隔离的关键步骤，其核心原理是 **Shell 上下文切换**。

- **修改 PATH 变量**：当执行激活脚本时，系统会动态修改**当前终端会话**的 PATH 环境变量。
- **优先级劫持**：脚本会将虚拟环境内部的路径置于 PATH 的最前面。这样，当你在终端输入 python 或 pip 命令时，系统会优先指向虚拟环境内部的路径，而非系统的全局路径。
- **环境标识**：激活成功后，命令行提示符通常会出现 (.venv) 字样，标志着当前的指令搜索路径已被锁定在局部环境内。

### 2.3 依赖库本地化存储
虚拟环境解决了多项目并存时的“依赖地狱”问题：
- **Site-packages 隔离**：在激活状态下使用 pip 安装的所有第三方包（如 FastAPI、Pydantic），都会被下载并存放在该环境特有的 site-packages 文件夹内。
- **防止环境腐烂**：这种机制确保了项目 A 依赖的库版本不会被项目 B 的安装操作所覆盖或破坏。通过这种方式，开发者可以在同一台机器上并存多个相互冲突的库版本。


## 3 依赖管理详解
依赖管理（Dependency Management）是在软件开发过程中，对项目所依赖的外部软件包（Packages/Libraries）进行识别、安装、版本约束及更新的系统化过程。

### 3.1 解决了什么问题
- **防止“环境腐烂”与版本冲突**：若项目 A 依赖某个库的 1.0 版本，而项目 B 偷偷将其升级到了 2.0 导致代码报错，依赖管理能通过锁定版本号来确保环境的稳定性。
- **解决传递依赖**：当你安装 FastAPI 时，它本身也依赖 Pydantic 和 Starlette。依赖管理工具会自动识别并安装这些“库所依赖的库”，无需你手动一个个查找安装。
- **实现“一键复现”**：当你把代码发给同事或部署到服务器时，对方不需要询问你装了什么，只需运行一行命令，系统就会根据清单还原出和你电脑上一模一样的运行环境。


### 3.2 在系统中的位置
- 逻辑位置：项目根目录下的配置文件中，常用文件名 requirements.txt。
- 物理位置：当执行安装命令后，这些库会被下载并存放在当前 Python 环境（.venv）的 Lib\site-packages 文件夹内。

### 3.3 怎么使用（示例）
- **安装需要的库**，安装 FastAPI 及其标准依赖
    ```bash
    pip install "fastapi[standard]"
    ```
- **导出依赖**，使用以下命令将当前环境的所有依赖及其精确版本号保存到文件中
    ```bash
    pip freeze > requirements.txt
    ```
    项目根目录下会生成 requirements.txt 文件，其中记录了依赖的库和版本号。
- **分发与还原**：如果你把代码发给同事，他只需拿到代码和这个 requirements.txt 文件，在自己的虚拟环境中运行
    ```bash
    pip install -r requirements.txt
    ```
    系统会自动读取文件，把里面记录的所有库及其指定版本全部安装好，确保他的环境和你的一致。


## 4 pip vs. uv
Python 工程开发中，pip 和 uv 是处理依赖管理和包安装的两大核心工具。

### 4.1 核心定位与定义
- ``pip``：是 Python 官方默认的包管理工具。用于执行安装、卸载及导出（如生成 requirements.txt）第三方库的操作。
- ``uv``：是由 Astral 团队开发的新一代 Python 一体化工具链。它不仅是一个包管理器，还集成了环境管理和项目管理能力，旨在替代 pip、virtualenv 及 pip-tools 等分散的工具。

### 4.2 关键特性对比
| 特性      |pip                               | uv                                              | 
| ---------|-----------------------------------|-------------------------------------------------|
| 开发语言  | Python                            | Rust（高性能基础）                               |
| 执行速度  | 较慢，涉及下载、解析和安装的完整周期 | 极快，通常比 pip 快 10~100 倍                     |
| 功能范围  | 仅限于包管理                       |  全栈管理：包含包管理、虚拟环境管理、Python 版本管理 |
| 环境管理  | 不负责虚拟环境，需配合 venv 模块使用 | 原生内置虚拟环境创建与切换功能                      |
| 依赖锁定  | 依赖锁定机制相对不完善              | 具有更现代、更快速的依赖解析与锁定能力               |

### 4.3 为什么需要 uv (解决 pip 的痛点)
- **安装效率低下**：在处理大型项目或大量依赖时，pip 的下载和依赖解析耗时较长。
- **工具碎片化**：一个完整的 Python 项目通常需要 pip 管理包、venv 管理环境、以及其他工具管理 Python 版本。uv 提供了一体化方案，避免了“工具太多”的问题。
- 环境独立性：pip 自身不具备环境隔离能力，若不小心在全局环境运行，容易导致依赖冲突。

### 4.4 常用操作

- 使用 pip 的标准流
    ```bash
    # 安装库
    pip install fastapi
    # 导出依赖清单
    pip freeze > requirements.txt
    # 卸载
    pip uninstall fastapi
    ```
    - 查看帮助文档：``pip --help``

- 使用 uv 的进阶流
    - 安装 uv (Windows/PowerShell/vs code 终端)
        ```bash
        powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
        ``` 
    - 重启 PowerShell
    - 验证：``uv --version``
    ```bash
    # 创建项目
    uv init rag-learning

    # 创建虚拟环境
    uv venv
    # 等价于
    python -m venv .venv

    # 安装依赖
    uv add langchain
    # 等价于
    pip install langchain

    # 删除依赖
    uv remove langchain

    # 同步依赖，自动安装项目全部依赖。
    uv sync

    # 运行脚本
    uv run main.py
    ```
