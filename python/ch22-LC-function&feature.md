# 逻辑核心 (Logical Core)--函数与高级特性


## 1 函数基础：逻辑的封装与复用
- 函数是组织好的、可重复使用的代码段，旨在实现单一或相关联的功能，是提高代码模块性的核心工具。
- 函数分为 Python 内建函数，如 print() 和 用户自定义函数。

### 1.1 自定义函数规则
- 定义方式：以 ``def`` 关键字开头，后接函数名和圆括号 ()，函数内容以冒号 : 开始并缩进。
    ```python
    def 函数名（参数列表）:
        函数体
    ```  
- **返回值**：使用 return 结束函数并返回结果；若无 return 或未带表达式，则默认返回 None。
- **文档字符串**：函数第一行通常可选择性地使用字符串进行功能说明，方便后期维护。
- [log_execution_info 函数示例](../codes/python_base/app/ef/log_decorator.py)

### 1.2 参数传递哲学
在 python 中，类型属于对象，对象有不同类型的区分，变量没有类型，例如：
```python
a="test"
b=[1,2,3]
```
- [1,2,3] 是 List 类型，"test" 是 String 类型
- 变量 a 没有类型，它仅仅是一个对象的引用（指针），可以指向 List 类型对象，也可指向 String 类型对象。

**对象引用传递**：Python 中变量仅是对象的引用。
- **不可变类型**（如数字、字符串、元组）的传递类似于 “C++ 的值传递”，在函数内修改不会影响原对象。
  - 示例：fun(a)，传递的只是 a 的值，没有影响 a 对象本身。如果在 fun(a) 内部修改 a 的值，则是新生成一个 a 的对象。

- **可变类型**（如列表、字典）的传递类似于 “C++ 的引用传递”，在函数内部的修改会同步影响外部对象。
  - 示例：fun(b)，则是将 b 真正的传过去，修改后 fun 外部的 b 也会受影响。

### 1.3 灵活的参数类型
- **必需参数**：以正确的顺序传入函数，调用时的数量必须和声明时的一样。
- **关键字参数**：使用关键字参数允许函数调用时参数的顺序与声明时不一致，因为 Python 解释器能够用参数名匹配参数值。
- **默认参数**：调用函数时，如果没有传递参数，则使用默认参数。
- **不定长参数**：处理比当初声明时更多的参数，基本语法：
    ```python
    def functionname([formal_args,] *var_args_tuple ):
        "函数_文档字符串"
        function_suite
        return [expression]
    ```
    - 使用 ``*args`` 参数（一个*号），会以**元组(tuple)**的形式导入，存放所有未命名的变量参数。如果在函数调用时没有指定参数，它就是一个空元组。
    - 使用 ``**kwargs`` 参数（两个*号），会以**字典**的形式导入。


## 2 匿名函数：Lambda 表达式
对于一些逻辑简单、仅需临时使用的功能，Python 提供了 lambda 表达式来实现，不需要使用 def 语句定义一个标准函数。

- **特性**：其主体是一个表达式而非代码块，仅能封装有限的逻辑。
- **命名空间**：lambda 拥有独立的命名空间，不能访问参数列表外或全局命名空间里的参数。

- lambda 表达式**语法**：
    ```python
    lambda [arg1 [,arg2,.....argn]]:expression
    ```

- **应用场景**：
  - **临时性的简单计算**：当你需要一个函数来执行一次性的逻辑，且不值得为其专门起名（使用 def）时。
  - **高阶函数的参数**：常作为参数传递给 sort()、filter() 或 map() 等函数，用于定义自定义的排序规则或过滤条件。
  - **数据清洗与转换**：在处理列表或字典等数据结构时，快速定义数据项的提取逻辑。
  - **回调函数**：在一些需要简短回调的逻辑中充当占位符。

## 3* 高级特性：装饰器 (Decorator)
装饰器是 Python 工程化开发中的“高级兵器”，其核心思想是 **用一个函数"包装"另一个函数**。

- 核心逻辑：在不修改原函数代码的前提下，动态扩展函数或类的功能。其语法是通过 @decorator_name 应用在目标定义之前。
- Python 提供了一些内置装饰器，如 @staticmethod 和 @classmethod。
- 示例：
    ```python
    def my_decorator(func):
        def wrapper(*args, **kwargs):
            print("执行前")
            func(*args, **kwargs)
            print("执行后")
        return wrapper

    @my_decorator
    def greet(name):
        print(f"Hello, {name}!")

    greet("Alice")
    ```
  示例说明：
  - @ 语法本质是函数替换
  - wrapper 才是真正执行的函数
  - 推荐使用 *args, **kwargs 提高通用性
  - 支持函数、类、甚至带参数的装饰器


**工程化应用场景**：
- 日志记录：自动记录函数调用信息、参数和返回值。[log_decorator](../codes/python_base/app/ef/log_decorator.py)
- 性能统计：统计函数执行时间
- 权限控制：限制函数访问权限
- 缓存：缓存函数结果，提高性能


## 4* 现代工程演进：类型提示与自动化
在 FastAPI 等现代框架中，函数特性与 类型提示 (Type Hints) 深度结合，使开发效率得到了质的飞跃。

- **声明式驱动**：开发者只需在函数参数中声明标准的 Python 类型，FastAPI 就能自动接管数据校验（利用 Pydantic）、数据转换（网络数据转 Python 对象）以及自动生成交互式文档（Swagger UI）。
- **依赖注入 (Depends)**：利用函数作为依赖项，可以轻松实现逻辑解耦（如安全认证、数据库连接管理），这是构建大型稳健 API 的核心机制。
- **异步并发**：通过 async def 声明函数，配合 await 机制，可以让函数在高性能 ASGI 服务器（如 Uvicorn）上以非阻塞方式运行，实现极高的并发处理能力。
