# 模块四：IO 与文件处理 (File Processing)--系统与标准库


## 1 Python3 标准库
### 1.1 内置电池 vs.标准库

- **核心联系：哲学与实现**
  
    - **标准库是“物理实体”**：它是指随 Python 解释器一起安装、位于系统目录（如 Python314\Lib）下的丰富模块集合。
    - **“内置电池”是“设计理念”**：这是个生动的比喻，就像一个自带电池的电子产品，用户拿到手后 **无需额外购买（安装第三方库）** 即可立即投入使用，完成大部分日常编程任务。

- **主要区别**  
    | 维度    | 标准库 (Standard Library)                 | 内置电池 (Batteries Included)                      |
    | -------- | ----------------------------------------| -------------------------------------------------|
    | 定义角度 | 侧重于软件组成。指那些经过官方测试、随发行版分发的模块 | 侧重于用户体验。强调 Python 解决问题的“开箱即用”能力  |
    | 存在形式 | 具体的文件和目录，如 os、sys 模块等        | 一种品牌特色和开发文化，体现了 Python 简单高效的本质  |
    | 涵盖范围 | 包含从底层操作系统接口到高级互联网访问的所有内建模块 | 泛指整个系统提供的、让开发者能快速上手开发应用的功能组合 |

### 1.2 标准库模块
Python 提供的内置模块涵盖了从底层系统到高层应用的各个方面，例如：

- **操作系统接口**：
  - **os 模块**：提供了许多与操作系统交互的函数，如创建、移动和删除文件和目录，以及访问环境变量等。
  - **sys 模块**：提供了与 Python 解释器和系统相关的功能，如解释器的版本和路径，以及与 stdin、stdout 和 stderr 相关的信息。

- **数据格式处理**：
  - **json 模块**：提供了 JSON 编码和解码函数，可以将 Python 对象转换为 JSON 格式，并从 JSON 格式中解析出 Python 对象。
  - **re 模块**：提供了正则表达式处理函数，可以用于文本搜索、替换、分割等。

- **时间与数学计算**：
  - **time 模块**：提供了处理时间的函数，如获取当前时间、格式化日期和时间、计时等。
  - **datetime 模块**：提供了更高级的日期和时间处理函数，如处理时区、计算时间差、计算日期差等。
  - **random 模块**：提供了生成随机数的函数，如生成随机整数、浮点数、序列等。
  - **math 模块**：提供了数学函数，如三角函数、对数函数、指数函数、常数等。

- **互联网访问**：
  - **urllib 模块**：提供了访问网页和处理 URL 的功能，包括下载文件、发送 POST 请求、处理 cookies 等。

- **工程基础设施**：如用于实现环境隔离的 ``venv`` 模块，它是现代 Python 工程化的地基。


## 2 os 模块
### 2.1 核心功能与应用场景
os 模块的主要职责是充当 Python 程序与底层操作系统之间的桥梁。

- **文件与目录管理**：提供了创建、移动、重命名和删除文件或目录的函数。如，在自动化脚本或 Web 应用中处理用户上传的文档时，经常需要调用这些接口进行物理存储管理。
- **环境变量访问**：通过 ``os.environ`` 访问系统环境变量。这是实现 **“代码与配置分离”** 的关键技术。开发者可以将敏感信息（如 API 密钥、数据库密码）存储在系统环境变量中，而不是硬编码在 .py 文件里，从而提高程序的安全性和灵活性。
- **系统信息获取**：获取当前工作目录、更改目录或执行特定的系统指令。

### 2.2 工程实践
- **推荐的导入风格**：建议使用 ``import os`` 而非 from os import *。
  - 原因：防止 os 模块中随操作系统变化的函数（如 os.open()）覆盖 Python 内部的同名内置函数 open()，从而避免引发难以察觉的 Bug。

- 在使用 os 这样的大型模块时内置的 dir() 和 help() 函数非常有用：
  - 使用 ``dir(os)``：可以快速查看该模块支持的所有函数列表。
  - 使用 ``help(os)``：查阅详尽的帮助手册和每个函数的具体用法说明。

- **配合资源管理**：在涉及文件描述符等系统资源的操作时，务必配合 with 语句使用，以实现“预定义的清理操作”

## 3 datetime 模块
datetime 模块是标准库中用于处理日期和时间的高级工具包。相比于基础的 time 模块，它提供了更面向对象的接口，能够处理更复杂的业务逻辑。核心能力包括：
- **高级日期处理**：不仅能获取当前时间，还能进行日期和时间的加减运算。
- **计算时间差/日期差**：支持通过特定对象（如 timedelta）计算两个时间点之间的间隔。
- **时区处理**：提供了处理不同地理区域时间差异的能力，这在分布式 Web 系统中至关重要。

主要核心类：
- ``datetime`` 类：最常用的类，同时包含日期和时间信息（年、月、日、时、分、秒）。
- ``date`` 类：仅处理日期信息（年、月、日）。
- ``time`` 类：仅处理时间信息（时、分、秒、微秒）。
- ``timedelta`` 类：用于表示两个时间点之间的时间间隔（如“3 天 5 小时”），是进行日期算术运算的关键。
