## 1 迭代器 (Iterator)
迭代器 (Iterator) 是处理序列数据的高级工具。它不仅是一种遍历手段，更是实现高性能、低内存占用程序的关键机制。

### 1.1 核心定义与本质
- **基本概念**：迭代器是一个可以记住遍历位置的对象。
- **设计哲学**——**“惰性求值”**：迭代器在访问元素时才计算值，而不是一次性将所有数据加载到内存中。这种机制在处理超大型数据集或无限序列时，能极大提升内存效率。
- **统一遍历接口**：它为列表、元组、字典等各种容器提供了一种通用的访问方式。

### 1.2 迭代器协议
一个对象要成为迭代器，必须实现 Python 定义的两个特殊方法：
- ``__iter__()``：返回迭代器对象本身。
- ``__next__()``：返回序列中的下一个元素。
    - 当没有更多元素可供访问时，它会抛出 StopIteration 异常，以告知循环结构停止遍历

在实际操作中，可以使用内置函数进行手动控制：
- 使用 ``iter(obj)`` 获取一个迭代器对象。
- 使用 ```next(it)``` 获取下一个值。

### 1.3 常用内置迭代工具
在日常开发中，如下工具通过迭代器机制提供了优雅的逻辑控制：
- range()：生成一个整数序列，常用于 for 循环。它是一个典型的不在内存中创建完整列表的迭代工具。
- enumerate()：在遍历序列时，同时获取元素的索引和值。


## 2* 生成器 (Generator)
- 生成器(Generator) 是使用了 ``yield`` 关键字 定义的特殊函数，是迭代器的一种特殊且高度简化的实现方式。
- 它通过 **“惰性求值”** 机制，在处理海量数据、流式任务或复杂的资源管理时，表现出极佳的性能和内存效率

### 2.1 暂停与恢复机制
- **非立即执行**：当调用一个生成器函数时，它不会立即执行函数体，而是返回一个**生成器对象**。
- **挂载状态**：每当对生成器调用 ``next()`` 时，函数会运行到 ``yield`` 语句处并 **“暂停”**。此时，函数会返回 ``yield`` 后的值，并保存所有当前的运行状态（包括局部变量和执行位置）。
- **断点续传**：下次调用 ``next()`` 时，函数从上次暂停的地方继续执行，直到遇到下一个 ``yield`` 或函数结束。

### 2.2 生成器的显著优势
相比于传统的列表或手动编写的迭代器类，生成器具有以下工程价值：

- **内存极度节省**：生成器不会一次性将所有元素加载到内存中，而是按需产生数据。这使其成为处理无限序列、大型日志文件或 TB 级数据库记录的唯一选择。
- **编写效率高**：编写普通迭代器需要手动定义类并实现 ``__iter__()`` 和 ``__next__()`` 方法；而生成器只需一个函数，Python 会自动处理所有迭代协议逻辑并维护执行上下文。
- **代码简洁**：它将循环逻辑和状态维护压缩在一个函数内，避免了样板代码。

### 2.3 迭代器 vs. 生成器
| 特性       | 迭代器 (Iterator)                  | 生成器 (Generator)                              |
| ---------- | ---------------------------------- | ----------------------------------------------- |
| 实现方式   | 通常通过类实现，需手动定义协议方法 | 通过包含 yield 的函数或生成器表达式定义         |
| 状态维护   | 需手动维护内部状态（如索引计数器） | 自动维护函数执行的上下文和变量                  |
| 代码复杂度 | 较多，逻辑分散                     | 极简，逻辑集中                                  |
| 关系       | 是一类广义的对象                   | 是迭代器的一种具体简化实现                      |
| 典型用途   | 封装复杂的遍历逻辑                 | 快速创建节省内存的数据序列，或实现协程/异步逻辑 |

### 2.4 现代工程应用：FastAPI 中的 yield
在现代 Web 框架 FastAPI 中，生成器的逻辑被进一步延伸到依赖注入 (Depends) 系统中：

- **资源生命周期管理**：你可以创建一个带有 yield 的函数作为依赖项（例如数据库连接）。
- **自动清理机制**：FastAPI 会在处理请求前运行到 yield 处（开启连接），在请求完成后（无论成功还是报错）从 yield 之后继续执行（关闭连接）。这实际上利用了生成器的暂停特性实现了一种异步上下文管理模式。

**实践建议**：在处理任何可能产生大量数据的循环时，应优先考虑 “生成器表达式” 或 yield 函数。在构建 Web 服务（如 FastAPI）时，利用 yield 进行依赖项管理是确保资源（如数据库、缓存、文件句柄）安全释放的最佳实践。
