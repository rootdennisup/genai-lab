## 1 列表 (List)：灵活的序列与算法基础
- 列表是 Python 中最通用的有序、可变序列，支持异质元素和切片操作。
- 列表是实现基本数据结构（如堆栈和队列）的首选工具。

- **常用方法**：
    - ``append(x)``：在末尾追加元素。
    - ``pop([i])``：移除并返回指定位置（默认为末尾）的元素。
    - ``sort()``：原地对列表进行排序。

- **数据结构实现**：
    - **堆栈 (LIFO)**：利用 ``append()`` 和 ``pop()`` 可以非常高效地实现“后进先出”逻辑。
    - **队列 (FIFO)**：虽然可以用列表实现，但在开头插入或删除性能较差（O(n)）。工程中推荐使用 collections.deque（双端队列），其两端操作的时间复杂度均为 O(1)。


## 2 元组 (Tuple)：不可变记录与安全性
- 元组是不可变的有序序列，一旦创建无法修改，这确保了数据的“只读”安全性。
- 通常用于存储具有归属关系的**异质数据集**。

**应用场景**：
- 作为字典的 **键(Key)**，因为元组是不可变的。
- 作为函数返回多个值的载体。
- 存储固定结构的记录（如经纬度坐标）。


## 3 字典 (Dictionary)：结构化数据与 JSON 的基石
- 字典是**键值对 (Key-Value)** 的映射结构，是处理结构化数据的核心。
- 从 Python 3.7 起，字典会保持插入顺序。
- 字典是模拟 JSON 对象 的直接载体。在 Web 框架（如 FastAPI）中，前端传来的数据在后端通常表现为字典或字典的嵌套。
- 键必须是不可变类型（如字符串、数字或元组）且在同一个字典中唯一。


## 4 集合 (Set)：唯一性与数学运算
集合是一个无序且元素唯一的容器。

- 核心用途：
    - 去重操作：快速从列表或元组中剔除重复元素。
    - 集合运算：支持交集 (&)、并集 (|)、差集 (-) 等数学运算，适合处理用户权限交叉或推荐系统逻辑。

注意：创建一个空集合必须使用 set()，因为 {} 默认创建的是空字典。


## 5 列表推导式 (List Comprehensions)
- 列表推导式是一种从一个或多个迭代器快速构建序列（如列表、字典、集合）的简洁语法。
- **语法本质**：它将传统的 for 循环和 if 条件判断压缩到一行代码中。
- **基本格式**：``[表达式 for 变量 in 迭代器 if 条件]``。

- **核心优势**：
  - **代码简洁**：原本需要 3-5 行的循环操作（初始化列表、循环、判断、append），现在只需一行即可完成。
  - **可读性更强**：能更清晰地表达“根据现有数据生成什么样的新数据”的意图。
  - **性能优异**：在底层执行效率通常比手动编写 list.append() 循环略高。

- **对比示例**：
  - 传统方式：初始化列表 -> 循环 -> 判断 -> append
    ```python
    new_list = []
    for x in range(10):
        if x % 2 == 0:
            new_list.append(x**2)
    ```

  - 推导式方式：[表达式 for 变量 in 迭代器 if 条件]
    ```python
    new_list = [x**2 for x in range(10) if x % 2 == 0]
    ```


## 6 嵌套结构 (Nested Structures)
嵌套结构是指将一个数据结构作为另一个数据结构的元素，例如“列表中的列表”（二维矩阵）或“字典中的列表”（复杂的 JSON 对象）。

- **应用场景**：
  - **模拟复杂业务数据**：在实际工程（如 FastAPI 应用）中，从前端接收到的数据往往是多层嵌套的 JSON。
  - **层级化组织**：方便表达具有归属关系的数据，例如“一个用户包含多条订单，每条订单包含多个商品”。
  - **精准校验**：借助 Pydantic 等库，可以对深层嵌套的每一个字段进行类型校验。

- 具体案例：
```python
# 嵌套结构案例：字典中嵌套列表
user_data = {
    "username": "coder_api",
    "roles": ["admin", "developer"], # 字典的值是一个列表
    "projects": [
        {"id": 1, "name": "FastAPI 学习"}, # 列表里又嵌套了字典
        {"id": 2, "name": "Pytest 自动化"}
    ]
}

# 访问嵌套数据
# 获取第一个项目的名称
first_project_name = user_data["projects"]["name"]
print(f"第一个项目名称: {first_project_name}")

# 结合推导式处理嵌套结构
# 提取所有项目的 ID 列表
project_ids = [p["id"] for p in user_data["projects"]]
print(f"所有项目 ID: {project_ids}")
```


## 7 核心容器应用总结

### 7.1 工程化演进：类型提示驱动
现代 Python 工程将这些容器与**类型提示 (Type Hints)** 深度结合：

- **自动序列化**：Python 的内置类型（如 str、int、float、bool、list 等）可以被自动转换为网络传输的 JSON 格式，简化了 API 的开发流程。
- **验证与安全**：Pydantic 作为“运行时门卫”，利用 Rust 编写的核心逻辑对这些复杂的数据结构进行极速校验，确保进入业务逻辑的数据百分之百符合预期的“形状”。

### 7.2 应用总结

- 核心容器提供了数据的存储方案，在进行数据建模时，可以按如下原则：
  - 频繁查找用字典
  - 保证唯一用集合
  - 防止篡改用元组
  - 顺序处理用列表

- 列表推导式提升了数据的处理效率，嵌套结构则解决了复杂业务逻辑的建模问题。
  
这些特性的结合使 Python 在 Web API 开发和数据分析领域具备了极高的生产力。
