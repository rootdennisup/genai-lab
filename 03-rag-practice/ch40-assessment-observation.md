# 评估与观测阶段

在 RAG（检索增强生成）系统的生命周期中，评估与观测是确保其从实验原型走向生产就绪的核心环节。

## 上下文召回率（Context Recall）
上下文召回率（Context Recall） 衡量的是**检索结果的完整性**，即检索器是否成功找回了所有能够回答用户问题的关键信息。

- **计算逻辑**：在检索到的前 k 个文档中，找到的相关文档占数据集中所有真实相关文档总数的比例。
- **评估意义**：高召回率意味着系统能够成功召回大部分关键背景知识，减少因信息缺失导致的回答不全。

## 上下文精确率（Context Precision）
上下文精确率（Context Precision） 衡量**检索结果的准确性**，侧重于评估检索到的内容中噪声的多寡以及排名质量。

- 计算逻辑：衡量在检索到的前 k 个文档中相关文档所占的比例，以及与参考答案相关的条目是否被排在靠前的位置。
- 评估意义：高精确率意味着检索结果中的噪声较少，且最相关的知识能被优先推给大模型，这有助于提升生成的质量。


## 忠实度 / 可信度（Faithfulness）
忠实度（Faithfulness，也称作 Groundedness） 衡量生成的答案是否完全**基于检索到的上下文信息**。

- **评估目标**：量化大模型的生成内容是否“忠于”参考知识，而不是依赖模型自身的预训练知识进行“自由发挥”。
- **实现方法**：将生成的答案分解为一系列独立的声明（Claims），然后在提供的上下文中验证每一个声明的真伪。最终分数是所有被上下文证实的声明所占的比例。


## 幻觉（Hallucination）
幻觉（Hallucination）是指大模型输出的结果不完全正确甚至完全错误（即“一本正经地胡说八道”）的现象。

- 产生原因：包括训练知识偏差、过度泛化推理、理解局限性以及缺乏特定垂直领域知识。
- 评估关系：忠实度（Faithfulness）指标正是为了量化幻觉的程度而设计的。RAG 架构诞生的初衷之一就是为了通过引入外部实时知识来尽可能解决大模型的幻觉问题。

## 观测与跟踪（Tracing）
观测与跟踪（Tracing）是为了打破大模型应用的“黑盒”状态，记录应用运行时的底层细节，以便进行调试和调优。

- **框架内部机制**：LlamaIndex 提供了 LlamaDebugHandler 等回调类，可以记录关键事件（如文本分割、嵌入、检索、大模型调用）的执行步骤、时间戳、输入信息和输出内容。
- **第三方平台**：企业级应用通常集成 Langfuse 或 LangSmith 等平台。这些平台支持端到端的链路追踪，开发者可以直观地查看每一次大模型调用的完整 Prompt、检索出的原始 Node 以及响应耗时，从而快速定位问题根源。

通过 “RAG 评估三元组”（上下文相关性、忠实度、答案相关性）结合全链路的 Tracing 观测，开发者可以科学地量化 RAG 应用的性能并持续优化。
